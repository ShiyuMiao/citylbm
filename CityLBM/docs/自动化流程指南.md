# CityLBM 自动化流程指南

## 概述

CityLBM 支持三种运行模式，从完全手动到完全自动化：

| 模式 | 说明 | 需要条件 |
|------|------|---------|
| **Mode 0** | 仅生成 Case 文件 | 无 |
| **Mode 1** | 自动部署 + 编译 + 运行 | FluidX3D 源码 + MSBuild |
| **Mode 2** | 仅部署文件 | FluidX3D 源码 |

---

## Mode 0: 仅生成 Case 文件（手动集成）

### 使用场景
- 首次使用，尚未安装 FluidX3D
- 需要手动调整参数后再运行
- 想要了解完整的工作流程

### 操作步骤

1. **在 Grasshopper 中设置场景**
   - 使用 `Create Scene` 组件创建场景
   - 使用 `Add Buildings` 添加建筑几何体
   - 使用 `Generate Grid` 生成计算网格

2. **运行 Run Simulation 组件**
   - Mode 选择 `0`（默认）
   - Run 设为 `True`
   - 不需要填写 FluidX3D Path

3. **查看输出**
   - Case Dir: Case 文件目录
   - Message: 详细的手动操作步骤

4. **手动部署到 FluidX3D**

   **方法 A: 使用自动化脚本**
   ```powershell
   cd CityLBM\tools
   .\deploy_fluidx3d.ps1 -CaseDir "<Case目录>" -FluidX3DPath "<FluidX3D目录>" -Mode 0
   ```

   **方法 B: 手动复制文件**
   ```
   复制 setup.cpp → <FluidX3D>/src/setup.cpp
   复制 defines.hpp → <FluidX3D>/src/defines.hpp
   复制 buildings.stl → <FluidX3D>/buildings.stl
   创建 <FluidX3D>/output/ 目录
   ```

5. **编译运行**
   ```bash
   # Visual Studio
   打开 FluidX3D.sln → Build → Release x64 → 运行
   
   # 或命令行
   msbuild FluidX3D.sln /p:Configuration=Release /p:Platform=x64
   ./bin/Release/x64/FluidX3D.exe
   ```

## Mode 1: 自动部署 + 编译 + 运行（全自动化）

### 使用场景
- 已安装 FluidX3D 和 Visual Studio
- 需要频繁运行模拟
- 追求最高效率

### 前置条件

1. **安装 FluidX3D**
   ```bash
   git clone https://github.com/ProjectPhysX/FluidX3D.git
   cd FluidX3D
   ```

2. **安装 Visual Studio** (Windows)
   - 下载 Visual Studio 2022 Community（免费）
   - 安装时勾选"使用 C++ 的桌面开发"

3. **配置路径**
   - 在 GH 组件中填写 FluidX3D Path
   - 或修改 `citylbm.config` 文件

### 操作步骤

1. **在 Run Simulation 组件中设置**
   ```
   Scene        = [连接场景输出]
   Grid         = [连接网格输出]
   FluidX3D Path = D:\FluidX3D  (您的 FluidX3D 目录)
   Mode         = 1
   Run          = True
   ```

2. **等待完成**
   - 组件会自动执行：
     1. 生成 Case 文件
     2. 部署到 FluidX3D
     3. 编译项目
     4. 运行模拟
     5. 收集 VTK 结果到 Case 目录

3. **读取结果**
   - VTK 文件自动复制到 `<Case目录>/output/`
   - 使用 `Read VTK` 组件读取

## Mode 2: 仅部署文件（手动编译运行）

### 使用场景
- 需要调整编译参数
- 想要在不同配置下测试
- 需要检查生成的代码

### 操作步骤

1. **在 Run Simulation 组件中设置**
   ```
   FluidX3D Path = D:\FluidX3D
   Mode         = 2
   Run          = True
   ```

2. **手动编译运行**
   ```bash
   cd D:\FluidX3D
   msbuild FluidX3D.sln /p:Configuration=Release
   ./bin/Release/x64/FluidX3D.exe
   ```

## 命令行工具

### deploy_fluidx3d.ps1

**位置**: `CityLBM/tools/deploy_fluidx3d.ps1`

**用法**:
```powershell
# 仅部署
.\deploy_fluidx3d.ps1 -CaseDir "C:\Temp\CityLBM\MyScene" -FluidX3DPath "D:\FluidX3D" -Mode 0

# 部署 + 编译
.\deploy_fluidx3d.ps1 -CaseDir "C:\Temp\CityLBM\MyScene" -FluidX3DPath "D:\FluidX3D" -Mode 1

# 完整流程
.\deploy_fluidx3d.ps1 -CaseDir "C:\Temp\CityLBM\MyScene" -FluidX3DPath "D:\FluidX3D" -Mode 2
```

### run_simulation.bat

**位置**: `CityLBM/tools/run_simulation.bat`

这是一个简化的批处理文件，可以双击运行。

**配置**:
1. 右键编辑 `run_simulation.bat`
2. 修改以下两行：
   ```batch
   set "CASE_DIR=C:\Users\MSY\AppData\Local\Temp\CityLBM\CityLBM Scene"
   set "FLUIDX3D_PATH=D:\FluidX3D"
   ```
3. 保存后双击运行

## 完整工作流示例

### 场景 1: 快速验证

```
1. Rhino 创建简单建筑
2. Grasshopper 连接 CityLBM 组件
3. Run Simulation (Mode 0) → 生成 Case
4. 检查生成的文件
5. 手动运行 FluidX3D 验证
```

### 场景 2: 批量模拟

```
1. 配置 FluidX3D 路径
2. 创建多个场景变体
3. 使用 Mode 1 自动运行
4. 批量读取 VTK 结果
5. 对比分析
```

### 场景 3: 参数研究

```
1. 使用 Mode 2 仅部署
2. 手动调整 defines.hpp 参数
3. 重新编译运行
4. 记录不同参数的结果
```

## 目录结构

### Case 目录
```
<场景名>/
├── setup.cpp          # FluidX3D 主程序
├── defines.hpp        # 宏定义和参数
├── buildings.stl      # 建筑几何体
├── output/            # VTK 结果文件（运行后生成）
│   ├── result_500.vtk
│   ├── result_1000.vtk
│   └── ...
└── run_simulation.bat # 快捷运行脚本
```

### FluidX3D 目录
```
FluidX3D/
├── src/
│   ├── setup.cpp      # 被 CityLBM 覆盖
│   ├── defines.hpp    # 被 CityLBM 覆盖
│   └── lbm.hpp
├── .citylbm_backup/   # 原始文件备份
│   ├── setup.cpp.original
│   └── defines.hpp.original
├── output/            # VTK 输出目录
├── buildings.stl      # 场景几何体
└── FluidX3D.sln
```

## 故障排除

### 编译失败

**问题**: 找不到 MSBuild
**解决**: 
- 安装 Visual Studio 2022
- 或安装 Build Tools for Visual Studio

**问题**: CUDA 错误
**解决**:
- 确保 NVIDIA 显卡驱动最新
- 安装 CUDA Toolkit 11.x 或 12.x

### 运行失败

**问题**: 找不到 buildings.stl
**解决**: 确保 STL 文件在 FluidX3D 根目录

**问题**: VTK 输出为空
**解决**: 检查模拟参数是否合理（网格、时间步等）

### 结果读取失败

**问题**: Read VTK 组件无法读取
**解决**:
- 确认 VTK 文件存在
- 检查文件路径是否正确
- VTK 文件格式应为 ASCII
